# Audio-navigation reproduction

This example is the public, redistributable counterpart to the private camera proof shown in the
Build Week demo. It is intentionally synthetic so anyone can inspect the source and regenerate it.

The 24-second video stays visually static in an ordinary `3x3` overview. A short audio transient
near 12 seconds proposes a candidate range; a brief pink marker appears just afterward and is
visible when the agent zooms into that range. Audio does not identify the marker. It only supplies
an address worth inspecting.

## Generate the sample

Requirements: `ffmpeg` and `ffprobe` on `PATH`.

```bash
./examples/audio-navigation/generate-sample.sh
```

The script writes `examples/audio-navigation/sample.mp4`. The committed MP4 is generated by this
same script.

## Run the overview

```bash
npx -y @getsquish/squish@0.3.1 \
  examples/audio-navigation/sample.mp4 \
  --json \
  --out /tmp/squish-audio-overview
```

Give the returned sheet to a vision model and ask:

> The ordinary visual samples look static. Which audio interval deserves a closer look?

Treat the audio band as a candidate only. On this fixture, inspect approximately `11.5–13.5s`.

## Zoom and verify

```bash
npx -y @getsquish/squish@0.3.1 \
  examples/audio-navigation/sample.mp4 \
  --density 6x6 \
  --start 11.5 \
  --end 13.5 \
  --json \
  --out /tmp/squish-audio-zoom
```

The dense sheet confirms the brief pink marker visually. The source fixture intentionally makes
no semantic claim about the audio. Public `0.3.1` normalizes activity across this complete source
clip; it does not compare separate video files on a shared scale.
